# ポッドキャストエピソード台本生成

ai-castポッドキャストの台本を生成して `ai-cast/podcast-gen/content/` に保存する。

## ポッドキャストの方向性

Backend × Infrastructure を軸にしたテック系ポッドキャスト。主な対象領域:
- AWS、Terraform、IaC
- システム設計、可用性、信頼性
- データベース設計・運用
- バックエンドアーキテクチャ全般

## 手順

### 1. エピソード番号の決定

`ai-cast/podcast-gen/content/` 内の既存ファイルを確認し、次のエピソード番号を採番する（ゼロ埋め3桁: 001, 002, ...）。

### 2. 既存エピソードの確認

`ai-cast/podcast-gen/content/` の既存台本をすべて読み、過去にどのトピックを扱ったか把握する。同じトピック・切り口の重複を避ける。

### 3. トピックの発見

上記の対象領域からWebSearchで**今話題になっているトピック**を探す。

検索ソース（毎回異なる組み合わせで探すこと）:
- Zenn トレンド
- はてなブックマーク テクノロジー
- AWS / GCP / Azure 公式ブログの最新アップデート
- HashiCorp公式ブログ
- Reddit r/aws, r/devops, r/database
- Hacker News
- X (Twitter) のエンジニア界隈の議論
- 各種カンファレンスの最新セッション

**多様性を保つルール:**
- 過去エピソードと同じトピック・同じ切り口は避ける
- 毎回異なるサブ領域から選ぶ（AWS→DB→IaC→設計→可用性のようにローテーション）
- 「最新ニュース系」「深掘り解説系」「比較系」「失敗談/アンチパターン系」など切り口も変える

候補を2〜3個、それぞれの切り口と一緒に挙げてユーザーに選んでもらう。

### 4. ソース記事の収集

選ばれたトピックについて、ソースとなる記事を3〜5本収集し、内容を読み込んで台本の土台にする。

### 5. 台本の生成

以下のフォーマットで台本を生成する。既存エピソード `ai-cast/podcast-gen/content/001-claude-code-skills.md` を参考にすること。

**フォーマット:**

```markdown
# エピソード: {タイトル}

## メタ情報
- 想定尺: 約5分
- 形式: MC（進行役）+ ゲスト（詳しい人）の対話
- ソース: {調査で見つけた記事の概要}

---

## 台本

MC: ...

ゲスト: ...

（MC/ゲストの対話形式で、自然な会話として構成）

---

## 参照記事
1. [記事タイトル](URL) - 著者名
```

**台本のルール:**
- 話者は必ず `MC:` と `ゲスト:` のプレフィックスで始める（TTS multi-speaker用）
- MCは進行役で質問を投げ、ゲストが詳しく解説する構成
- 約5分の想定尺（読み上げで5分程度のテキスト量）
- 専門用語は噛み砕いて説明し、具体例を交える
- 冒頭でテーマ紹介、最後にまとめがある自然な流れ

### 6. ファイル保存

`ai-cast/podcast-gen/content/{番号}-{slug}.md` に保存する。
slugはトピックから英語のkebab-caseで生成する。

### 7. 次のステップの案内

保存完了後、以下を案内する:

```
台本を確認して、OKなら以下で音声を生成できます:
npx tsx main.ts audio content/{ファイル名}
```
