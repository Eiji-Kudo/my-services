aicastのX(Twitter)投稿用カルーセル画像を、HTML+CSSで作成する。ブラウザで開いて1枚ずつスクショして投稿する。

## 構成

- ファイル構成: slide1.html, slide2.html（各スライド独立） + styles.css（共通CSS） + script.js（ハッシュ表示制御） + index.html（iframe一覧）
- スライドサイズ: 1350×1080px
- index.htmlは全スライドをiframeで縦に並べて表示
- 各スライドHTMLは単体でも開ける（styles.css, script.jsを読み込み）
- URLハッシュ(#slide1等)で単体表示対応
- 外部フォント/画像なし。アイコンはSVGまたはCSS図形

## デザイン方針

- ダークモード: 背景 #000、カード #1c1c1e
- アクセントカラー: purple (#bf5af2)、グラデーション (#bf5af2 → #ff375f)
- フォント: system-ui（日本語が読みやすいタイポ、余白多め、行間広め、見出し強め）
- 高級感あるシンプルデザイン。小さすぎない文字（スマホで読める）
- 全スライドに統一感（同じグリッド、同じ余白 --pad: 64px）
- ブランドロゴ "aicast" は左上に紫ドット付きで各スライド共通配置
- スライド番号は右下に薄く表示（例: 01 — 02）

## スライド構成（全2枚）

### Slide 1: フック + 仕組み

上半分: テキスト + iPhoneモックアップ2台を横並び
- 左: テキストエリア
  - 見出し（56px太字）: あなた専用の"学びPodcast"を自動生成。
    - "学びPodcast" にグラデーションテキスト効果
  - サブ（22px）: 興味に合わせてオーダーメイドで、5分に編集して届ける。
- 右: iPhoneモックアップ2台
  - iPhone 1: X Import画面
    - 𝕏ロゴ + 「連携中」バッジ
    - 「タイムラインから生成中...」サブテキスト
    - トピックリスト5件（完了ドット/スピナー/未着手の3状態）
      - 各アイテム: ドット状態 + トピック名 + ハンドル情報 + 所要時間
    - フッター: 「2 / 5 エピソード生成済み」プログレス表示
  - iPhone 2: Now Playing画面
    - 「再生中」ヘッダー
    - アルバムアート（紫グラデーション正方形、タイトル表示）
    - 曲名 + アーティスト（aicast · 5分）
    - プログレスバー + 再生時間
    - 再生コントロール（前へ/再生/次へ）

下半分: 3ステップ横並びフロー
- STEP 01「興味を取り込む」: X連携アイコン(SVG) + チップ(𝕏 Feed, Claude Code, Infra) + 説明
- STEP 02「情報を集めて編集」: フィルターアイコン(SVG) + チップ(15ソース → 3トピック) + 説明
- STEP 03「短尺で届く」: ヘッドホンアイコン(SVG) + 波形ビジュアライゼーション + 説明
- ステップ間は破線矢印で接続

最下部: ボトムロー
- 「寝る前 / 散歩 / 通勤で、読む代わりに追いつく。」テキスト
- ピル型バッジ2つ: 「レベル調整（初心者〜実務）」「要点→背景→次アクション」

### Slide 2: エピソード例

- 見出し（40px太字）: こんな回が"あなた向け"に作られる
- 2×2グリッドで4つの例を表示（角丸カード、ダーク背景）
  - Example A: Claude Code アップデート解説（今週版）
    - 何が変わった？（要点3つ）/ 何が嬉しい？（開発フローへの効き）/ すぐ試す手順（最短）/ つまずき所（注意点）
  - Example B: ChatGPT / Claude / Gemini "Skills"使い分け
    - 何が得意？（タスク別）/ 失敗パターン（よくある罠）/ 使えるプロンプト型（テンプレ）/ チーム導入のコツ
  - Example C: インフラ新機能まとめ（AWS/GCP/Cloudflare）
    - 重要アップデートだけ抽出 / 影響（コスト/安定性/運用） / 使うべき条件・使わない条件 / 次に追うべきリンク
  - Example D: Big Tech 求人ブリーフィング（職種別）
    - 今出てるロールの傾向 / 求められるスキルセット / 書類で刺さる実績の書き方 / 面接で見られるポイント（想定）
- 下部注記: "あなたの興味×あなたのレベル"で編集する想定　／　※プロトタイプ検証中
